# Public Website i18n — `custosoft-workers`

> Das Worker-Backend liefert die Marketing-Seite (custosoft.de),
> Impressum / Datenschutz / AGB / Widerruf und die Bewerbungs-Landing
> in mehreren Sprachen. Aktuell: **Deutsch + Englisch**.

---

## Schnell-Übersicht

```
src/i18n/
├── locales.ts        # Locale-Type, Erkennung, Cookie
├── strings.ts        # Alle UI-Strings (Landing + Nav + Footer) als T-Map
├── nav.ts            # commonNav(locale) + commonFooter(locale) inkl. Sprachschalter
├── shared-style.ts   # CSS — locale-unabhängig
├── landing.ts        # buildLandingHtml(locale): string
├── legal.ts          # defaultImpressum/Datenschutz/AGB/Widerruf(locale)
└── legal-page.ts     # legalPageHtml(title, content, locale)

src/routes/web-public.ts   # Routes — locale-aware mit /:locale/-Prefix
```

## URL-Schema

| URL | Was |
|---|---|
| `/` | Auto-Erkennung Sprache → 302 zu `/de/` oder `/en/` |
| `/de/` | Deutsche Landingpage |
| `/en/` | Englische Landingpage |
| `/de/impressum`, `/en/imprint` | Impressum |
| `/de/datenschutz`, `/en/privacy` | Datenschutzerklärung |
| `/de/agb`, `/en/terms` | AGB / Terms of Service |
| `/de/widerruf`, `/en/withdrawal` | Widerrufsbelehrung |
| `/lang/de`, `/lang/en` | Setzt Cookie + leitet zurück (Sprachschalter im Footer) |
| `/impressum`, `/datenschutz`, … | Legacy: leiten auf bevorzugte Sprache um (SEO/Bookmarks) |
| `/apply/:code` | Bewerbungs-Landing (Public-Job-Link) — locale-aware |
| `/beta-signup` (POST) | Beta-Anmeldung — Antwort + E-Mail in der Sprache des Browsers |
| `/beta-count` (GET) | JSON, sprachneutral |

## Spracherkennung — Reihenfolge

1. **Cookie** `lang=de|en` (vom Footer-Switcher gesetzt)
2. **Pfad-Prefix** (`/de/...` oder `/en/...`)
3. **Accept-Language** Header (erster matchender Locale-Code)
4. **DEFAULT_LOCALE** (`de`)

→ Implementiert in [`detectLocale()`](src/i18n/locales.ts).

## Eine neue Sprache hinzufügen

### Schritt 1 — Locale-Code registrieren

In [`src/i18n/locales.ts`](src/i18n/locales.ts):

```ts
export const LOCALES = ['de', 'en', 'fr'] as const          // 'fr' anhängen
export const LOCALE_LABEL: Record<Locale, string> = {
  de: 'Deutsch', en: 'English', fr: 'Français',             // Native Anzeige
}
export const LOCALE_HTML_TAG: Record<Locale, string> = {
  de: 'de', en: 'en', fr: 'fr',                             // <html lang="…">
}
```

TypeScript zeigt dir jetzt überall, wo Übersetzungen fehlen — Compile schlägt fehl bis du sie ergänzt hast.

### Schritt 2 — UI-Strings übersetzen

In [`src/i18n/strings.ts`](src/i18n/strings.ts) jede `STRINGS`-Eintrag um den neuen Locale-Code erweitern:

```ts
nav_start: { de: 'Start', en: 'Home', fr: 'Accueil' },
…
```

### Schritt 3 — Legal-Pages übersetzen

In [`src/i18n/legal.ts`](src/i18n/legal.ts):

1. `LEGAL_SLUGS[fr] = { impressum: 'mentions-legales', datenschutz: 'confidentialite', agb: 'conditions', widerruf: 'retractation' }` ergänzen.
2. `LEGAL_TITLES.{impressum,datenschutz,agb,widerruf}.fr` befüllen.
3. Pro Funktion (`defaultImpressum`, `defaultDatenschutz`, `defaultAGB`, `defaultWiderruf`) einen `case 'fr': return frenchHtml()` hinzufügen.

> **Hinweis Impressum:** Das deutsche Impressum bleibt rechtsverbindlich (§ 5 TMG). Andere Sprachen sind höflichkeitsübersetzungen — der Hinweis-Banner oben verlinkt deshalb auf `/de/impressum` als Original.

### Schritt 4 — `wrangler dev` und prüfen

```bash
cd custosoft-workers
npm run dev
```

→ `http://localhost:8787/fr/` zeigt die französische Landing-Page.

### Schritt 5 — Production deploy

```bash
npm run deploy
```

## Strings-Map verstehen

[`strings.ts`](src/i18n/strings.ts) sieht so aus:

```ts
export const STRINGS = {
  hero_h1_em:   { de: 'Team.',       en: 'Team.' },
  hero_sub:     { de: 'Stempeluhr · …', en: 'Punch clock · …' },
  …
} as const

export function t(key: StringKey, locale: Locale, ...args: string[]): string { … }
```

Aufrufen:

```ts
t('hero_sub', 'en')                           // → "Punch clock · …"
t('beta_count_label_html', 'de', '100')       // → "von <strong>100</strong> Beta-Plätzen vergeben"
```

Platzhalter `{0}`, `{1}` werden durch die Args ersetzt. Reihenfolge ist beim Übersetzen frei wählbar.

## Lifecycle-Hinweise

- **Cookie** wird mit `Max-Age=31536000` gesetzt — bleibt 1 Jahr.
- **Legacy-URLs** (`/impressum`, `/datenschutz`, `/agb`, `/widerruf`) bleiben für immer als 302-Redirects bestehen — SEO und alte Bookmarks brechen nicht.
- **D1-Override** ist locale-aware: ein Eintrag in `legal_pages` (slug, locale, content) gilt nur für genau diese Sprache. Wenn ein deutscher Override vorhanden ist, sehen englische Besucher trotzdem den englischen Default aus `legal.ts` — kein Bleeding zwischen Sprachen.

  **Erste Migration nötig** auf laufenden DBs (das alte Schema kannte nur `slug` als unique):
  ```bash
  cd custosoft-workers
  wrangler d1 execute custosoft-db \
    --file=scripts/update_legal_pages_locale.sql --remote
  ```
  Bestehende Einträge werden automatisch als `locale='de'` markiert. Die Routen funktionieren auch vor der Migration weiter — der Worker erkennt das alte Schema und behandelt slug-Einträge dann nur noch für `de`.
- **Beta-Signup** verwendet die erkannte Sprache für Antwort + E-Mail.
- **Apply-Page** (`/apply/:code`) folgt der Browser-Sprache.

## Häufige Stolpersteine

**„Mein neuer Eintrag in `STRINGS` wird im JS-Block der Landing-Page nicht übersetzt."**

→ Prüfe ob der Wert ohne `<` oder `'` Zeichen ist die durch `JSON.stringify()` für die Browser-JS-Block-Injektion noch escaped werden müssen. Sieh dir `beta_count_label_html` in `landing.ts` als Beispiel an — dort wird `JSON.stringify(t(...))` verwendet.

**„Impressum auf Englisch — ist das rechtlich sicher?"**

→ Ja: Wir zeigen einen Banner oben („The legally binding version is the German one") und linken zu `/de/impressum`. Damit erfüllen wir die TMG-Pflicht (deutsches Impressum vorhanden) und bieten gleichzeitig nicht-deutschen Besuchern eine verständliche Übersicht.

**„Sprache wechselt nicht obwohl ich den Cookie gesetzt habe."**

→ Die Erkennung priorisiert Cookie über Pfad-Prefix. Wenn dein Browser den Cookie nicht akzeptiert (z.B. third-party cookies disabled), greift der Pfad-Prefix. Beide Mechanismen funktionieren parallel.

**„Type-Error: ‚Property 'fr' is missing'."**

→ Genau dafür ist das `STRINGS as const` da. TypeScript zwingt dich, jeden neuen Locale komplett zu übersetzen, bevor das Projekt baut.
